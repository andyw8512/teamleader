# Changelog

## 0.1.4

- 增加“子 agent 模型选择规则”：简单任务创建子 agent 时，必须优先使用 `gpt-5.3-codex-spark`；只有特别复杂任务才允许继承组长模型。

## 0.1.3

- 增加“不停推进原则”：除非目标完成、失败终止、等待用户批准或外部依赖不可用，否则组长不得停住。
- 要求每次唤醒至少执行一个明确推进动作，例如复核、集成、续派、验证或明确阻塞。
- 明确 `DONT_NOTIFY` 只能表示不打扰用户，不能表示没有内部推进动作。
- 强化所有子 agent 完成后的主线程接棒规则，避免停在“已完成列表”旁边空转。

## 0.1.2

- 强化 heartbeat 提示词规范，明确禁止把心跳写成“检查 / 识别 / 汇总 / 建议下一步”为主的检查型提示词。
- 要求 heartbeat 的主动作必须是“立即继续推进、立即续派、立即复核、立即集成”。
- 增加“心跳提示词禁忌”说明，减少组长退化成检查员而不是调度器的情况。
- 同步更新当前 `回撤优化心跳` automation 的提示词为强推进版本。

## 0.1.1

- 增强心跳检测生命周期管理。
- 明确要求在项目完成、终止、取消或被替代时主动关闭原心跳。
- 为心跳机制补充关闭条件、`已关闭` 状态和交付前关闭自检。
- 避免任务结束后仍然保留每分钟提醒。

## 0.1.0

- 首次发布 `teamleader` skill 仓库版本。
- 提供 `leader` 技能定义文件和 `openai` 元数据。
- 强化“心跳检测 / 每分钟巡检”要求为执行闸门：
  - 未设定并确认生效时不得进入执行阶段。
  - 缺少 `心跳检测机制` 或 `循环任务已设置确认` 时视为计划无效。
  - 增加计划输出前自检清单，减少组长漏写心跳机制的概率。
- 补充仓库 `README`，说明安装、验证和使用方式。
